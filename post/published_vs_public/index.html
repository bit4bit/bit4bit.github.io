¿Público y/o Públicado?

Cuando ré/factorizamos surgen nuevos elementos de los cuales es importante determinar cuales son públicos y y cuales son publicados, si no hacemos esta aclaración podemos fácilmente transgredir conceptos, veamos lo un ejemplo.

class Billetera
  def initialize
    @movimientos = []
  end
  
  def ingreso(concepto, valor)
    @movimientos << [concepto, valor]
  end

  def gasto(concepto, valor)
    @movimientos << [concepto, -valor]
  end

  def saldo
    @movimientos.inject(0) { |saldo, movimiento| saldo + movimiento[1] }
  end
end

La clase Billetera es pública y publicada (concepto común al asumir que cualquiera puede referenciar la clase y hacer uso de esta) ya que esperemos que sea usada.

Ahora realicemos una refactorización buscando la apertura del movimiento.

class Movimiento
  attr_reader :valor
  
  def initialize(concepto, valor)
    @concepto = concepto
    @valor = valor
  end
  
  def balancear(saldo)
    saldo.balancear(@valor)
  end
end

class Saldo
  attr_reader :resultado
  
  def initialize
    @resultado = 0
  end

  def balancear(valor)
    @resultado += valor
  end
  
  def aplicar(movimientos)
    movimientos.each_with_object(self) { |movimiento, saldo| movimiento.balancear(saldo) }
  end
end

class Billetera
  def initialize
    @movimientos = []
  end
  
  def ingreso(concepto, valor)
    @movimientos << Movimiento.new(concepto, valor)
  end
  
  def gasto(concepto, valor)
    @movimientos << Movimiento.new(concepto, -valor)
  end

  def saldo
    Saldo.new.aplicar(@movimientos).resultado
  end
end

Las clases Balance, Saldo, Movimiento son por defecto públicas al menos en Ruby (cada lenguaje tiene un comportamiento por defecto por ejemplo en C# el alcance por defecto es private) ahora las clases deben estar publicadas para crear acoplamiento desde otras secciones del programa? no, no están publicadas a excepción de Balance, actualmente son elementos que surgieron de la refactorización y tienen alta cohesión con el origen de la creación (Balance).
