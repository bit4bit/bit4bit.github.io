redchan

16/Feb 2015 2 min. read

RedChan Deslocalizando los canales de Go con Redis. TUTORIAL Para esto necesitamos instalar el paquete glivo. go get github.com/bit4bit/redchan INICIALIZACION Para crear un canal usando Redis como transporte usamos la estructura RedisChannel{“uuid”, 0} , siendo el primer parametro el identificador unico del canal usado tanto para recibir como para enviar, el segundo parametro es el tamano de Buffer antes de bloquear la gorutina. # Creacion de un canal sin buffer redisChannel := RedisChannel{"uuid", 0} # Creacion de un canal con buffer redisChannel := RedisChannel{"uuid", 10} # Al finalizar el uso del canal lo cerramos llamando *Close* defer redisChannel.Close() USO Ahora creamos los canales para enviar la data #buffered sendCh, sendErr := redchan.Send(redisChannel, 0) if sendErr != nil { t.Fatal(sendErr) } #unbuffered sendCh, sendErr := redchan.Send(redisChannel, 1) if sendErr != nil { t.Fatal(sendErr) } #Atencion con los parentesis sendCh() <- []byte("data") #Al finalizar el canal se notifica el receptor tal como los canales de *Go* close(sendCh()) Tambien podemos crear varios canales Go por un canal RedChan.

glivo

7/Aug 2014 1 min. read

GLIVO Sencillo kit de herramientas para el desarrollo aplicaciones IVR desde el lenguaje GO y Freeswitch haciendo uso del mod’ulo mod_event_socket. TUTORIAL Para esto necesitamos instalar el paquete glivo. $ go get github.com/bit4bit/glivo Inicializaci’on de Glivo El procedimiento general Glivo es crear un socket en la interfaz y puerto al que se comunicar’a el Freeswitch, para esto usamos la funci’on glivo.NewSession del paquete en conjunto con net.Listener o bien podemos directamente la funci’on glivo.Listen del paquete Glivo, miremos: glivo.NewSession package main import ( "net" "log" "github.com/bit4bit/glivo" ) func main() { listen, listenErr := net.Listen("tcp", "localhost:5050") if listenErr != nil { log.Fatal(listenErr) } socket = glivo.NewSession(listen) defer socket.Close() } glivo.Listen Nos permite abrir el la direcci’on TCP indicada directamente.
